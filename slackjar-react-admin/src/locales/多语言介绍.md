# 多语言国际化方案

## 📚 技术栈

本项目采用**双重国际化机制**，结合以下技术实现完整的多语言支持：

### 1. 业务文本国际化
- **i18next**: 核心国际化库
- **react-i18next**: React 绑定，提供 Hook 和组件
- **i18next-browser-languagedetector**: 浏览器语言自动检测

### 2. UI 组件库国际化
- **Ant Design ConfigProvider**: Ant Design 组件库的语言包切换
- **dayjs**: 日期时间库的国际化

### 3. 状态管理
- **Zustand**: 全局状态管理，持久化用户语言偏好

## 🔧 实现方式

### 1. i18next 初始化配置

**文件**: `src/locales/i18n.ts`

```typescript
import i18n from 'i18next';
import LanguageDetector from 'i18next-browser-languagedetector';
import {initReactI18next} from 'react-i18next';
import {resources} from "./resources";

i18n
    .use(LanguageDetector)        // 自动检测浏览器语言
    .use(initReactI18next)        // 集成到 React
    .init({
        fallbackLng: 'zh',        // 降级语言：中文
        lng: 'zh',                // 默认语言：中文
        debug: true,              // 开发模式调试
        resources: resources,     // 翻译资源
        interpolation: {
            escapeValue: false    // React 已处理 XSS，无需转义
        }
    });
```

**关键配置说明**：
- `LanguageDetector`: 自动检测浏览器语言，并记住用户选择（存储在 localStorage）
- `fallbackLng`: 当翻译缺失时使用的降级语言
- `debug`: 开发环境下会在控制台显示翻译键的警告信息

### 2. 翻译资源管理

**文件**: `src/locales/resources.ts`

```typescript
import en from "./translations/en.json";
import zh from "./translations/zh.json";

export const resources = {
    "en": { translation: en },
    "zh": { translation: zh }
}
```

**翻译文件格式** (`zh.json` / `en.json`):

```json
{
  "loading": "加载中...",
  "menu": {
    "home": "首页",
    "user management": "用户管理",
    "system management": "系统管理"
  }
}
```

### 3. 全局状态管理（Zustand）

**文件**: `src/store/appGlobalStore.ts`

```typescript
import {create} from 'zustand'
import {persist} from 'zustand/middleware'
import {Locale} from "antd/es/locale"
import zhCN from "antd/locale/zh_CN"
import enUS from "antd/locale/en_US"

export interface AppGlobalState {
    locale: Locale
    language: 'zh' | 'en'
    setLocale: (locale: Locale, language?: 'zh' | 'en') => void
}

// 从 localStorage 读取上次保存的语言设置
const savedLanguage = (localStorage.getItem('language') as 'zh' | 'en' | null) || 'zh'
const initialLocale = savedLanguage === 'en' ? enUS : zhCN

export const useAppGlobalStore = create<AppGlobalState>()(
    persist(
        (set) => ({
            locale: initialLocale,
            language: savedLanguage,
            setLocale: (locale: Locale, language?: 'zh' | 'en') => {
                set({
                    locale,
                    language: language || (locale === enUS ? 'en' : 'zh')
                })
            }
        }),
        {
            name: 'app-global-storage',
            partialize: (state) => ({language: state.language})
        }
    )
)
```

**核心功能**：
- 使用 `persist` 中间件自动持久化语言设置到 localStorage
- 启动时从 localStorage 恢复用户上次的语言选择
- 只持久化 `language` 字段，避免存储大型 locale 对象

## 💡 使用方式

### 1. 在组件中使用翻译

#### 基础用法

```typescript
import {useTranslation} from "react-i18next";

function MyComponent() {
    const {t, i18n} = useTranslation();
    
    return (
        <div>
            <h1>{t('menu.home')}</h1>
            <p>{t('loading')}</p>
        </div>
    );
}
```

#### 嵌套键访问

```typescript
// JSON 结构
{
  "menu": {
    "home": "首页"
  }
}

// 使用方式
t('menu.home')  // → "首页"
```

### 2. 切换语言

```typescript
import {useTranslation} from "react-i18next";
import {useAppGlobalStore} from "../store/appGlobalStore";
import enUS from 'antd/locale/en_US';
import zhCN from 'antd/locale/zh_CN';
import dayjs from 'dayjs';

function LanguageSwitcher() {
    const {i18n} = useTranslation();
    const {setLocale} = useAppGlobalStore();
    
    const switchToEnglish = () => {
        // 1. 切换 i18next 业务文本
        i18n.changeLanguage('en');
        
        // 2. 切换 dayjs 日期库语言
        dayjs.locale('en');
        
        // 3. 切换 Ant Design 组件语言（会自动持久化）
        setLocale(enUS, 'en');
    };
    
    const switchToChinese = () => {
        i18n.changeLanguage('zh');
        dayjs.locale('zh');
        setLocale(zhCN, 'zh');
    };
    
    return (
        <div>
            <button onClick={switchToEnglish}>English</button>
            <button onClick={switchToChinese}>中文</button>
        </div>
    );
}
```

### 3. 实际案例：顶部导航栏

**文件**: `src/layout/MyHeader/index.tsx`

```typescript
const {t, i18n} = useTranslation();
const {setLocale} = useAppGlobalStore();

const handlerChangeLang = (e: { key: string }) => {
    const language = e.key as 'zh' | 'en';
    const locale = language === 'en' ? enUS : zhCN;
    
    i18n.changeLanguage(language);     // 切换业务文本
    dayjs.locale(language);            // 切换日期格式
    setLocale(locale, language);       // 切换 UI 组件语言并持久化
}

const langItems: MenuProps['items'] = [
    {
        key: 'zh',
        label: "简体中文",
        onClick: handlerChangeLang
    },
    {
        key: 'en',
        label: "English",
        onClick: handlerChangeLang
    },
];
```

### 4. 路由菜单国际化

**文件**: `src/routers/router.tsx`

```typescript
const createMenuItems = (routers, key, t, topMenuOnly) => {
    return routers.map(item => ({
        key: key + processPath(item.path),
        icon: item.icon,
        label: t(item.label || item.path),  // 翻译菜单文本
        children: ...
    }));
}
```

当语言切换时，菜单会自动重新渲染并显示对应语言的文本。

### 5. 面包屑国际化

**文件**: `src/layout/Breadcrumbs/index.tsx`

```typescript
const {t, i18n} = useTranslation();

const breadcrumbs = useMemo(() => getBreadcrumbs(t, pathname), 
    [i18n.language, pathname]);
```

使用 `useMemo` 优化性能，只在语言或路径变化时重新计算。

## 🎯 工作流程

### 语言切换完整流程

```
用户点击语言切换按钮
    ↓
MyHeader.handlerChangeLang()
    ↓
1. i18n.changeLanguage('en')         → 切换业务文本翻译
2. dayjs.locale('en')                → 切换日期格式化语言
3. setLocale(enUS, 'en')             → 切换 Ant Design 语言包
    ↓
Zustand persist 中间件自动保存到 localStorage
    ↓
触发 React 重新渲染
    ↓
所有使用 t() 的组件自动更新文本
ConfigProvider 更新 antd 组件语言
路由菜单和面包屑自动刷新
```

### 应用启动流程

```
应用启动 (main.tsx)
    ↓
导入 i18n.ts，初始化 i18next
    ↓
LanguageDetector 检测浏览器语言或读取 localStorage
    ↓
App.tsx 从 Zustand store 读取语言配置
    ↓
Zustand persist 从 localStorage 恢复上次选择
    ↓
ConfigProvider 应用对应的 Ant Design 语言包
    ↓
渲染应用
```

## 🔍 调试技巧

### 1. 开启调试模式

在 `i18n.ts` 中设置 `debug: true`，会在控制台显示：
- 缺失的翻译键
- 语言切换事件
- 翻译解析过程

### 2. 检查 localStorage

打开浏览器开发者工具 → Application → Local Storage，查看：
- `i18nextLng`: i18next 保存的语言设置
- `app-global-storage`: Zustand 保存的全局状态

### 3. 验证翻译完整性

确保所有语言的 JSON 文件包含相同的键结构，避免运行时显示原始键名。

## ⚠️ 注意事项

### 1. 翻译键命名规范
- 使用小写字母和连字符：`user-management`
- 使用点号表示层级：`menu.home`
- 保持键名语义清晰

### 2. 避免硬编码文本
❌ **错误做法**:
```typescript
<h1>首页</h1>
```

✅ **正确做法**:
```typescript
<h1>{t('menu.home')}</h1>
```

### 3. 动态内容处理
对于包含变量的文本，使用插值：

```json
{
  "greeting": "你好，{{name}}！"
}
```

```typescript
t('greeting', { name: '张三' })  // → "你好，张三！"
```

### 4. 性能考虑
- 避免在循环中频繁调用 `t()`
- 使用 `useMemo` 缓存翻译结果
- 大型应用可配置翻译文件异步加载
